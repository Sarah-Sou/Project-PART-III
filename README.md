This project is my Part III submission for INFR 3120. It is a backend-driven Habit Tracker built with Node.js, Express, EJS templates, and MongoDB Atlas. The goal of this project was to create a functional, secure, and fully deployed web application that allows users to create, view, update, and delete habits while maintaining a clean structure and reliable server-side logic.

I originally explored building an Angular frontend, but I decided to remove it to keep the project consistent with the backend setup. The application now runs as a complete server-rendered site using EJS, which keeps everything simple, stable, and easy to maintain.

Features

User authentication (login, register, logout)

User Authentication: Sign up and Login using Local strategy.

Social Login: Support for Google, GitHub, and Discord authentication.

Ability to create, edit, and delete habits

Habit list page to view all habits

Profile page that supports updating the profile picture

Clean routing structure using Express

Fully deployed backend on Render

MongoDB Atlas hosted database
